How you designed the system

The main system is divided into 3 major parts. We have the server, client and GUI.

The server which can be found here
consist of 2 classes. The first class is called “ChatServer”, this can be located in the package Server.
This class is the main server, it performs 2 tasks.
The first task is to start the server at port 8081. The second task is to listen for clients that might connect to the server and assign a socket-Thread to each client.
By assigning a new socket-thread to each client, the server is able to handle multiple clients at once.
The second class which can be found in the Server package is “ServerThread”. This class has one job and that is to send a response to the clients request.

The client consists of 2 classes, Client and ClientThread.
Client have to get the username, then to start a ClientThread and afterwards listening through a scanner, when the user writes something.
For then to put it in a LinkedList, so the ClientThread will write the message to the server, next time it gets to the part in its while loop, that writes to the server. (Should happen almost instantly, unless there are A LOT of message to write from the server).
The ClientThreads job is to print and write between the client and the server, using a socket.
If the socket closes, the ClientThread should stop working aswell, also causing the Client to close down.

The GUI consists of 3 classes, the MainGUI which includes the visual parts of the GUI, a method to write into the chatbox, and a few eventlisteners doing something when triggered (e.g. sending a message).
The receiver, which constantly listens to the server for new messages, if there is a new message then appending it to the GUI chatbox.
And a sendGUI, which sends messages written in the GUI to the server client, attached to the GUIs socket.

How to use the system

How to use the system:

Download our chat client (GUI), write a name, and you're started

You can type 'msg:<*names*>:message', to type to specific people.

Otherwise you can just type something in, and the GUI will tell the server to write to all

Type logout: to logout

Without our GUI, you can call login:<*username*>, to set your name, note you can only do it once per client

You can write MSG:<*receivers*>:<*message*>, where you're splitting receivers by ','s. You can write to one, many or use '*' to write to all

You'll receive answers from the server in the format MSGRES:<*sender*>:<*message*>.

You'll get a client list everytime a person logs off or on the server.

Type 'logout:' to logout from the server
